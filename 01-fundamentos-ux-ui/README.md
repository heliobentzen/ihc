# 01 — Fundamentos de Experiência do Usuário (UX) e Design de Interface (UI)

> *"O design bom é o design invisível."* — Dieter Rams

---

## 🎯 Objetivos de Aprendizagem

Ao final deste módulo você será capaz de:
- Diferenciar UX de UI com exemplos concretos
- Compreender por que a experiência do usuário impacta negócios
- Identificar as disciplinas que compõem o guarda-chuva de UX
- Descrever o papel de cada profissional na equipe de produto

---

## 1. O Que é UX?

**UX (User Experience)** é a soma de todas as percepções, emoções e respostas de uma pessoa ao interagir — antes, durante e depois — com um produto, sistema ou serviço.

O termo foi popularizado por **Don Norman** no início dos anos 1990, quando trabalhava na Apple. Ele queria um nome que abrangesse *tudo* o que afeta a experiência da pessoa com o produto, além da simples interface.

### A metáfora do copo d'água 🥤

| Dimensão | Exemplo |
|----------|---------|
| **UX ruim** | Copo bonito, mas com borda afiada que corta o lábio |
| **UI ruim, UX ok** | Copo feio mas que encaixa perfeitamente na mão e não vaza |
| **UX + UI bons** | Copo ergonômico, esteticamente agradável e que não pinga |

---

## 2. O Que é UI?

**UI (User Interface)** é a camada visual e interativa através da qual o usuário se comunica com o produto. Engloba:

- Tipografia e hierarquia visual
- Paleta de cores e contraste
- Ícones e iconografia
- Layout e grade (grid)
- Animações e micro-interações
- Estados dos componentes (hover, focus, disabled, error…)

> **Analogia:** Se o UX é a estrutura e o encanamento de uma casa, o UI é a decoração e a pintura.

---

## 3. Diferenças Fundamentais

| Aspecto | UX | UI |
|---------|----|----|
| **Foco** | Jornada e satisfação do usuário | Aparência e interação visual |
| **Entregáveis** | Pesquisas, personas, fluxos, wireframes | Mockups, guias de estilo, assets |
| **Ferramentas** | Entrevistas, mapas de empatia, testes | Figma, Sketch, Adobe XD |
| **Pergunta central** | *O produto resolve o problema certo?* | *O produto é bonito e claro de usar?* |
| **Métrica** | Taxa de conclusão de tarefa, NPS | Consistência visual, taxa de erro |

## 5. Por Que UX Importa Para o Negócio?

### ROI de UX — Dados Reais

| Dado | Fonte |
|------|-------|
| Cada $1 investido em UX retorna $100 | Forrester Research |
| 88% dos usuários não retornam após uma má experiência | Sweor |
| Melhorar a taxa de conversão em 1% pode valer milhões | Nielsen Norman Group |
| Empresas líderes em CX superam o S&P 500 em 35% | Watermark Consulting |

### Exemplo: Redesign da Airbnb (2014)
A Airbnb quase faliu em 2009. Perceberam que as fotos dos imóveis eram horríveis. Contrataram fotógrafos profissionais → receita dobrou em semanas. **Insight:** UX vai além da tela.

## 6. O Modelo dos 5 Planos de Jesse James Garrett

Garrett definiu 5 camadas para pensar em UX, de abstrato para concreto:

```
5. SUPERFÍCIE    → Visual design, UI
4. ESQUELETO     → Layout, interface, navegação
3. ESTRUTURA     → Arquitetura de informação, design de interação
2. ESCOPO        → Requisitos funcionais, conteúdo
1. ESTRATÉGIA    → Necessidades do usuário, objetivos do negócio
```

> **Prática:** Analise um app que você usa todos os dias passando por cada um dos 5 planos.

## 7. Papéis em uma Equipe de Produto

| Papel | Responsabilidade principal |
|-------|---------------------------|
| **UX Researcher** | Conduz pesquisas para entender usuários |
| **UX Designer** | Projeta fluxos e experiências |
| **UI Designer** | Cria a interface visual |
| **Product Designer** | Cobre UX + UI + estratégia |
| **UX Writer** | Escreve textos de interface (microcopy) |
| **Interaction Designer** | Foca em animações e comportamentos |
| **Service Designer** | Projeta a experiência omnichannel |

## 🛠️ Práticas

### Prática 1 — Auditoria de UX/UI (30 min)
Escolha um app de e-commerce (ex: Mercado Livre, Amazon, Shopee).
1. Liste **3 problemas de UX** (fluxo quebrado, confusão, frustração)
2. Liste **3 problemas de UI** (contraste, hierarquia, espaçamento)
3. Para cada problema, proponha uma solução em 1 frase

### Prática 2 — Analogia da Porta (15 min)
Fotografe uma porta próxima a você. Responda:
- O design comunica claramente se devo puxar ou empurrar?
- Há affordance visual clara?
- O que mudaria para tornar o design mais intuitivo?

> Esta é a famosa "porta de Norman" — descrita no livro *The Design of Everyday Things*.

### Prática 3 — Comparativo de Apps (45 min)
Compare o onboarding de dois apps concorrentes (ex: Nubank vs. Itaú):
1. Cronômetro o tempo até a primeira ação útil
2. Conte o número de telas
3. Identifique o tom de voz (formal/informal)
4. Avalie a consistência visual
5. Produza uma tabela comparativa

## ✅ Checklist de Boas Práticas

- [ ] O produto tem um propósito claro?
- [ ] O usuário sabe onde está e para onde pode ir?
- [ ] As ações principais estão visíveis e acessíveis?
- [ ] O feedback de sistema é imediato e compreensível?
- [ ] A hierarquia visual guia o olhar para o que importa?
- [ ] O produto é consistente internamente?
- [ ] Os textos da interface são claros e sem jargão?
- [ ] O design funciona em diferentes tamanhos de tela?
